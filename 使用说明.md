# 内外网代码同步工具 使用说明

两个脚本配合，解决「外网写代码 → 内网部署」时内网独有文件不能外带的问题。

- `push_to_both.py`：**外网（个人电脑）** 用。一份代码双推到团队仓 + 个人仓，并自动把本次改动清单带进个人仓。
- `sync_code.py`：**内网** 用。下个人仓的 zip，自动按清单把改动文件覆盖到内网项目，内网独有文件不动。

仓库地址（已写进脚本默认配置）：

- 团队仓 `team`：`git@github.com:cangjie-ai/ark-agentic.git`
- 个人仓 `personal`：`git@github.com:YU-JI-KUI/ark-agentic.git`

---

## 一、外网：第一次配置（只做一次）

在你本地项目目录下：

```cmd
python push_to_both.py setup
```

它会给当前仓库加上 `team` 和 `personal` 两个 remote。

## 二、外网：日常双推

写完代码后，一条命令搞定：

```cmd
python push_to_both.py push --range HEAD~1..HEAD -m "feat: 改了意图识别"
```

`--range` 是**本次要同步到内网的改动范围**，由你决定（这是你选的「手动传范围」模式）：

- 最近 1 个提交：`HEAD~1..HEAD`（默认）
- 最近 3 个提交：`HEAD~3..HEAD`
- 指定两个提交之间：`abc1234..def5678`

`-m` 只在有未提交改动时需要（让脚本帮你提交代码）；如果你已经自己 commit 过，可以不带 `-m`。

脚本做的五步：

1. 提交代码（若有未提交改动）
2. 推送干净代码到 **团队仓**
3. 按 `--range` 生成 `changed_files.txt`
4. 把清单作为临时提交**强推到个人仓**
5. 本地回退临时提交，保持和团队仓一致

> 为什么这么绕：保证团队仓只有干净代码历史，`changed_files.txt` 只待在个人仓顶端。强推个人仓是安全的——它是你自己的仓，内网每次只读最新 zip。

## 三、内网：下载并同步

1. 在内网下载**个人仓** `ark-agentic` 的 zip（团队仓内网访问不到，所以下个人仓）。
2. 运行（第一次建议先 `--dry-run` 预览）：

```cmd
python sync_code.py --zip ark-agentic-main.zip --project D:\work\ark-agentic --dry-run
python sync_code.py --zip ark-agentic-main.zip --project D:\work\ark-agentic
```

不用手动指定清单——`sync_code.py` 会**自动读 zip 里自带的 `changed_files.txt`**。
也可以用 `--list 自己的清单.txt` 手动覆盖。

行为：

- 清单里普通行的文件**直接覆盖**到内网项目相同相对路径，缺目录自动建。
- 清单里 `[DEL]` 开头的文件会在内网**删除**：内网有就删，内网没有就跳过（不报错）。
- 内网项目里清单没提到的文件（你的内部独有代码）**完全不动**。
- 清单文件本身不会被复制进内网项目。
- 删除/覆盖后内网的 git 会显示所有改动，你 review 后再在内网提交即可。

## 完整流程图

```
外网个人电脑
  写代码
  python push_to_both.py push --range HEAD~1..HEAD -m "..."
        ├─ 干净代码 ──push──> 团队仓 cangjie-ai/ark-agentic
        └─ 代码+清单 ─强推─> 个人仓 YU-JI-KUI/ark-agentic
                                      │
内网                                  ▼ 下载 zip
  python sync_code.py --zip xxx.zip --project D:\work\ark-agentic
        └─ 按 zip 内 changed_files.txt 覆盖改动文件，内网独有文件不动
```

## 注意点

- 两个脚本都是纯 Python 标准库，无需 `pip install`。
- 命令均为 Windows CMD 兼容。
- `--range` 每轮只覆盖你指定的范围。如果某一轮你在内网漏同步了，记得下轮把范围拉长（比如 `HEAD~2..HEAD`），否则那次的文件不会进清单。
- 新增/修改/重命名的文件会**复制**；删除的文件会在内网**删除**（内网没有则跳过）。重命名会表现为「新增新名 + 删除旧名」。
