# 内外网代码同步工具 使用说明

两个脚本配合，解决「外网写代码 → 内网部署」时内网独有文件不能外带的问题。

- `push_to_both.py`：**外网（个人电脑）** 用。拉团队仓 master 最新代码，按指定 commit 生成改动清单，推到个人仓。
- `sync_code.py`：**内网** 用。下个人仓的 zip，按清单把改动文件覆盖/删除到内网项目，内网独有文件不动。

仓库地址（已写进脚本默认配置）：

- 团队仓 `team`：`git@github.com:cangjie-ai/ark-agentic.git`
- 个人仓 `personal`：`git@github.com:YU-JI-KUI/ark-agentic.git`

---

## 工作模型（先看这一节，理解了再用）

**核心：团队仓 master 就是 SSOT（事实之源）。**

- 你在外网写代码 → 走正常流程进团队仓 master（PR merge 或直接 push）
- 跑 `push_to_both.py sync` 时：
  - 本地强制对齐到 `team/master`（不在本地做任何代码改动）
  - 收集你指定的几个 commit 触碰过的所有文件路径作为候选
  - **以 team/master 当前状态决定动作**：
    - 候选文件在 master 上仍存在 → 标记为「复制」
    - 候选文件在 master 上不存在了 → 标记为「删除」
  - 清单作为一个临时提交强推到个人仓顶端，本地立刻回退
- 内网下个人仓 zip → 跑 `sync_code.py` → 按清单同步到内网项目

这套设计能让中间夹任意 commit（rename、revert、改名、cherry-pick）都不会算错——结果永远和 master 一致。

---

## 一、外网：第一次配置（只做一次）

在你本地项目目录下：

```cmd
python push_to_both.py setup
```

它会给当前仓库加上 `team` 和 `personal` 两个 remote。

## 二、外网：日常同步

### 前置条件

- 你的代码已经合入团队仓 master（PR merge 或直接 push 完毕）
- 本地工作区干净（没有未提交的改动）。**脏工作区会直接报错退出**，脚本不会擅自处理。

### 命令

```cmd
python push_to_both.py sync --commits <sha1> <sha2> <sha3> ...
```

`--commits` 是本次想同步到内网的所有 commit SHA：

- 可以传一个或多个
- 顺序无所谓
- 不需要连续
- 所有 commit **必须**已经在团队仓 master 历史里——否则脚本会报错

**举例**：你这一周往团队仓 master 推了 3 个不连续的 commit（中间还有别人的 commit 在 master 上），现在准备一次性同步到内网：

```cmd
python push_to_both.py sync --commits cd8a1a28 4c736d32 a9cf84a1
```

### 脚本做的五步

1. fetch team，本地 master `reset --hard` 到 `team/master`（要求工作区干净）
2. 校验每个 commit 都在 team/master 历史里
3. 收集这些 commit 触碰过的所有文件路径，对每个路径查 team/master 是否还有
4. 写 `changed_files.txt`，临时提交后强推到 `personal/master`
5. 本地 reset 回 team/master，保持干净

> 强推个人仓是预期行为——个人仓只是你自己的中转站，内网每次只读最新 zip。

### 清单文件 `changed_files.txt` 长什么样

```
# 本次改动清单，由 push_to_both.py 自动生成
# 普通行 = 复制到内网；[DEL] 开头 = 在内网删除该文件
# 基于团队仓 master @ 87fd6722
# 涉及 commit（动作以 team/master 为准：存在=复制、不存在=删除）:
#   cd8a1a28 refactor(memory): 完全插件化，core 零 plugins.memory 依赖
#   4c736d32 refactor(session/memory): 清理死代码、修复封装泄漏、合并冗余 API
#   a9cf84a1 feat: 多 POD 部署一致性 —— 持久层即 SSOT
src/ark_agentic/plugins/memory/manager.py
src/ark_agentic/core/session/compaction.py
...
[DEL] src/ark_agentic/core/memory/manager.py
[DEL] src/ark_agentic/core/memory/dream.py
...
```

## 三、内网：下载并同步

### 步骤

1. 在内网下载**个人仓** `ark-agentic` 的 zip（团队仓内网访问不到，所以下个人仓）
2. 第一次先 dry-run 预览：

```cmd
python sync_code.py --zip "C:\Downloads\ark-agentic-master*.zip" --project D:\work\ark-agentic --dry-run
```

3. 确认无误后真正执行：

```cmd
python sync_code.py --zip "C:\Downloads\ark-agentic-master*.zip" --project D:\work\ark-agentic
```

不用手动指定清单——`sync_code.py` 会**自动读 zip 根目录里的 `changed_files.txt`**。
也可以用 `--list 自己的清单.txt` 手动覆盖。

#### 关于 `--zip` 的通配符（强烈推荐用）

浏览器下第二份同名 zip 会自动改名成 `ark-agentic-master (1).zip`、`(2).zip`...
如果命令写死 `--zip ark-agentic-master.zip`，每次都得手动改名或改命令。

**用通配符一劳永逸**：`--zip "C:\Downloads\ark-agentic-master*.zip"`
- 匹配多个时，**按修改时间自动选最新的那个**（= 你刚下载的那份）
- 命令永远不用改，固定下来 copy-paste 就行
- 注意 Windows CMD 下要给 glob 加**双引号**，防止 shell 提前展开
- 多个匹配时日志会列出来给你看挑了哪个，不会"静默选错"

### 行为规则

- 清单里普通行的文件 → **复制覆盖**到内网项目相同相对路径，缺目录自动建
- 清单里 `[DEL]` 开头的文件 → 在内网**删除**
  - 内网原本有 → 真删除（日志会显示文件大小，方便核对）
  - 内网原本就没有 → **跳过，不报错**（这是预期行为，日志标 `[跳过删除]`）
- 内网项目里清单没提到的文件（内网独有代码）→ **完全不动**
- 清单文件本身（`changed_files.txt`）→ 不会被复制进内网项目

### 日志详细度

每个文件都有独立一行日志，分类如下：

```
[新增] some/new/file.py  (3.4 KB)        # 内网原本没有的文件被复制进来
[覆盖] some/existing/file.py  (12.1 KB)  # 内网原本有的文件被新版本覆盖
[删除] old/file.py  (5.6 KB)             # 内网原本有的文件被删除
[跳过删除] old/file.py                   # 清单要求删除但内网原本就没有 —— 正常
[缺失] some/file.py                      # 清单要求复制但 zip 里没有 —— 异常
```

执行最后会汇总：

```
将新增 12 个、将覆盖 68 个、将删除 9 个。
跳过删除（内网原本不存在）0 个、缺失（zip 里没有）0 个。
```

执行完，内网项目目录里的 `git status` 能看到所有改动，你 review 后再在内网提交即可。

## 完整流程图

```
外网个人电脑
  写代码 → 走正常流程合入团队仓 master（PR / push）
  python push_to_both.py sync --commits sha1 sha2 sha3
        ├─ fetch team + 本地 reset --hard team/master
        ├─ 收集 commit 触碰过的文件，以 master 现状定动作（存在=复制 / 不存在=删除）
        └─ 代码 + 清单 ── 强推 ──> 个人仓 YU-JI-KUI/ark-agentic
                                      │
内网                                  ▼ 下载 zip
  python sync_code.py --zip xxx.zip --project D:\work\ark-agentic
        └─ 按 zip 内 changed_files.txt 覆盖/删除，内网独有文件不动
```

## 注意点

- 两个脚本都是纯 Python 标准库，无需 `pip install`
- 命令均为 Windows CMD 兼容
- `--commits` 每轮只处理你指定的范围。**如果某一轮你在内网漏同步了**，下轮把那些 commit SHA 一起传进来就行（顺序无所谓，重复也无所谓）
- rename 类改动会**自动正确处理**：旧路径会进 `[DEL]`，新路径进复制——`sync_code.py` 既复制了新文件、也删除了旧文件，内网状态和 master 一致
- 删除路径在内网不存在 → **跳过、不报错**（这是预期行为，例如内网项目本来就比外网更新过、或者那个文件本来就只在外网生成过）
